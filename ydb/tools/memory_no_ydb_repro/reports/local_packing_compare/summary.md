# local_packing_compare

Сравнение `local_numa0/local_numa1` для двух порядков CPU:
- `interleave`: потоки сначала разносятся по разным CCD
- `dense`: потоки плотно заполняют CCD по 8 ядер подряд

Ключевой результат: knee действительно определяется CCD-locality. При `dense` кривая растет ступенями по `8` потоков, то есть по мере подключения новых CCD. При `interleave` тот же рост размазан до `12` потоков, потому что первые 12 потоков сразу расходятся по 12 CCD сокета.

Практическая интерпретация:
- Один densely packed CCD дает примерно `41 GB/s`.
- Два densely packed CCD дают примерно `79-80 GB/s`.
- Три densely packed CCD дают примерно `109-110 GB/s`.
- Четыре densely packed CCD дают примерно `137 GB/s`.
- Шесть densely packed CCD дают примерно `177 GB/s`.
- После заполнения всех 12 CCD (`96` потоков) `dense` и `interleave` сходятся к одному NUMA ceiling `~213-214 GB/s`.
